# Quoridor Agent and Teacher

---

## Overview

Quoridor is a two-player strategy game played on a 9×9 grid where players must reach the opposing side while placing walls to block their opponent. Because the ruleset is less well-known than games like Chess, novice players often struggle to form effective strategies. This project builds a Convolutional Neural Network (CNN) that evaluates Quoridor board states and recommends moves, and a game analyzer tool that reviews a player's decisions after each game to help them improve.

The model is based on the AlphaZero architecture, with a shared CNN backbone that splits into two heads: a **policy head** that recommends which move to make, and a **value head** that estimates whether the current position is winning or losing. Training data was generated by simulating games between existing search-based agents (Minimax, Minimax with Alpha-Beta Pruning, Expectimax, and Monte Carlo Tree Search), recording every (state, action, outcome) tuple for supervised learning.

## Reproducing Results

### Step 1 — Generate training data

```bash
python3 run_data_generation.py
```

This simulates 10,000 games between search agents and saves the results to `game_data.json`. Each record contains the board state before a move, the action taken, and the eventual winner. Adjust `NUM_GAMES` and `processes` at the top of the file as needed. On an 8-core machine this takes approximately 4 hours.

### Step 2 — Train the model

```bash
python3 console/model/train.py
```

Trains the CNN for 20 epochs using the generated data. The best model is saved to `quoridor_model.pth` based on validation loss. Training on a GPU takes approximately 5 minutes for 10,000 games of data.

### Step 3 — Evaluate the model

```bash
python3 console/model/eval.py
```

Runs 100 games of the model against a random agent and Expectimax, then prompts to optionally run against Minimax with alpha-beta pruning. Reports win rate and value head F1 score for each opponent.

### Step 4 — Run the game analyzer

```bash
python3 analyzer.py
```

Choose `H` for human mode or `A` for auto mode (random agent plays for testing). After the game ends, an `analysis.txt` file is saved with a move-by-move breakdown. Open it in VS Code for proper board rendering with color.

### Step 5 — Play a human vs human game

```bash
python3 main.py
```

## Key Results

| Opponent | Win Rate | P1 Wins | P2 Wins | Value F1 |
|---|---|---|---|---|
| Random | 98% | 49/50 | 49/50 | 0.88 |
| Expectimax | 50% | 0/50 | 50/50 | 1.00 |
| Minimax (alpha-beta) | 50% | 0/50 | 50/50 | 1.00 |

The 50% win rate against search agents reflects a positional bias in the training data — the model wins every game as Player 2 and loses every game as Player 1. This is consistent with an imbalance in how often P2 won during data generation, which the model learned from. The value head F1 of 1.0 against both search agents indicates the model perfectly predicted the game outcome from each position during evaluation.
